### How Haar Cascade Works
The Haar Cascade classifier uses a set of positive and negative images to train a classifier. It relies on Haar-like features, which are essentially rectangular features that capture the difference in pixel intensity between adjacent regions. The algorithm works by scanning an image using a sliding window technique, where each window is classified as either containing the object (e.g., face) or not based on the learned features.

The process involves three main steps:

1. Haar Features: These are simple rectangular features used to represent objects (e.g., face).

2. Integral Image: The integral image allows the fast calculation of features at any scale.

3. Cascade of Classifiers: A series of increasingly complex classifiers that discard regions unlikely to contain an object at each step, improving efficiency.
### Steps to Train a Haar Cascade Classifier
Involves several steps: preparing a dataset, creating positive and negative image sets, and using OpenCV's tools to train the classifier

Positive Images: These images should contain the object (e.g., faces, cars, etc.) we want to detect. The object should be clearly visible and centered. These images are typically cropped to contain only the object of interest.

Negative Images: These images should not contain the object we want to detect. They will be used to teach the classifier what to avoid.

After preparing the dataset, we can use OpenCV's `opencv_traincascade` command-line tool to train the classifier. The training process involves specifying parameters such as the number of positive and negative samples, the number of stages in the cascade, and the feature type.

Then we can use the trained classifier in the following way:
```python
import cv2

# Load the trained Haar cascade classifier
face_cascade = cv2.CascadeClassifier('data_dir/cascade.xml')

# Read an image
img = cv2.imread('group_photo.jpg')
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

# Detect objects (e.g., faces) in the image
objects = face_cascade.detectMultiScale(gray, scaleFactor=1.1, minNeighbors=5, minSize=(30, 30))

# Draw rectangles around the detected objects
for (x, y, w, h) in objects:
    cv2.rectangle(img, (x, y), (x+w, y+h), (255, 0, 0), 2)

# Display the result
cv2.imshow('Detected Objects', img)
cv2.waitKey(0)
cv2.destroyAllWindows()
```

## The Light Bulb Moment
After hours of research and trying different approaches, I realised I am solving a solved problem, guess where? Google Photos. Google Photos has a feature that automatically recognises faces in photos and groups them together.

This is done using advanced machine learning techniques, specifically deep learning models like Convolutional Neural Networks (CNNs), I will not go into the details of CNNs here, but they are highly effective for image recognition tasks.

I found that there's already exists a Python library called `face_recognition` that wraps around dlib's state-of-the-art face recognition built with deep learning. It provides simple APIs to perform face detection and recognition tasks.

### The Miracle 10 Lines

```python
import face_recognition
import cv2

# Load and convert the image
image = cv2.imread("group_photo.jpg")
image_rgb = cv2.cvtColor(image, cv2.COLOR_BGR2RGB)

# Detect faces
face_locations = face_recognition.face_locations(image_rgb)

# Extract and save faces
for i, (top, right, bottom, left) in enumerate(face_locations):
    face_image = image[top:bottom, left:right]
    cv2.imwrite(f"face_{i+1}.jpg", face_image)
    cv2.rectangle(image, (left, top), (right, bottom), (0, 255, 0), 2)

cv2.imshow("Faces", image)
cv2.waitKey(0)
cv2.destroyAllWindows()
```

### How it Works?
Every face has unique features that can be captured and represented as a high-dimensional vector, often referred to as an embedding. The `face_recognition` library uses a pre-trained deep learning model to extract these embeddings from images.

The vectors of each face can be represented as points in a high-dimensional space. 
![Face Embeddings](files/blog-contents/blog-2026-01-17-12-00-00/img-2.jpg)

If vectors (embeddings) of two faces are close to each other in this space, it indicates that the faces are similar or likely belong to the same person. Conversely, if the vectors are far apart, the faces are likely different.

We use Euclidean Distance to measure the similarity between two face embeddings. The Euclidean distance between two points in a multi-dimensional space is calculated using the formula:

$ d(p, q) = \sqrt{\sum_{i=1}^{n} (p_i - q_i)^2} $
